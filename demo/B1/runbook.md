# Runbook
## Инструкция по старту и общему взаимодействию с сервисом
Перед началом перейдите в корень проекта и выполните команду чтобы скопировать env переменные
```bash
cp .env.example .env
```

Чтобы поднять платформу выполните команду
```bash
docker compose up -d
```
Для скачивания сторонних образов (redis, postgres, metabase итд) при первом запуске потребуется интернет

После старта на 80 порту будет работать nginx, который распределяет запросы по сервисам
* http://localhost - платформа A/B тестирования (главный сервис, решение кейса)
* http://localhost/emulator - система-эмулятор как отдельный сервис в докер-контейнере. Эмулирует внешнюю систему с пользователями, на которых проводится тест. Нужна для прогона сложных сценариев (тест подсчета метрик, guardrails и так далее) с разными вероятностями разных событий. По сути система эмулятор - это многофункциональный скрипт для генерации данных по конфигу.
* http://localhost/metabase - UI визуализация метрик, доп фича

Все сервисы внутри общаются через docker сеть.

Для генерации тестовых данных используются python скрипты которые вызывают api, их надо запускать локально.
